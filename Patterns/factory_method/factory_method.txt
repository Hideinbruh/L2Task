Паттерн Фабричный метод (Factory Method) позволяет создавать объекты определенного типа, но делегирует процесс
создания на подклассы, которые могут изменять тип создаваемого объекта.

Применимость:
- Когда нужно создавать объекты определенного типа, но неизвестно заранее, какой именно тип будет создан
- Когда нужно делегировать процесс создания объектов на подклассы

Плюсы:
- Гибкость и расширяемость системы
- Уменьшение связанности между объектами
- Возможность изменять тип создаваемого объекта без изменения кода клиента

Минусы:
- Усложнение структуры программы из-за большего количества классов
- Увеличение времени выполнения программы из-за необходимости создания дополнительных объектов

Примеры использования на практике:
- Создание объектов в играх
- Создание объектов в GUI приложениях
- Создание объектов в фреймворках
